# Описание проекта MeteoRW

## Общее описание
Проект MeteoRW представляет собой распределенную систему для управления метеорологическими и железнодорожными данными. Система состоит из нескольких микросервисов, взаимодействующих между собой для обеспечения комплексного управления климатическими и транспортными данными.

## Микросервисы

### 1. Meteostation
- **Описание**: Микросервис для управления метеостанциями
- **Основные функции**:
  - Создание, чтение, обновление и удаление метеостанций
  - Поиск станций по geohash
  - Автоматическая генерация geohash на основе координат
  - Валидация данных
- **API**: http://localhost:8081/api
- **Технологии**: Go, Gin, GORM, PostgreSQL
- **Структура данных**:
  - Метеостанции с координатами, высотой и geohash
  - Валидация данных через специальный валидатор

### 2. Meteodata
- **Описание**: Микросервис для управления метеорологическими измерениями
- **Основные функции**:
  - Хранение регулярных метеорологических измерений
  - Хранение данных о свете (LED)
  - Хранение данных о TTTR
  - Хранение данных о снеге (SNOW)
- **API**: http://localhost:8083/api
- **Технологии**: Go, Gin, GORM, TimescaleDB
- **Структура данных**:
  - Регулярные измерения (видимость, коды погоды, ветер, осадки, температуры)
  - Данные о свете, TTTR и снеге

### 3. RailWayStation
- **Описание**: Микросервис для управления железнодорожными станциями
- **Основные функции**:
  - Создание, чтение, обновление и удаление железнодорожных станций
  - Управление районами, линиями и областями
- **API**: http://localhost:8082/api
- **Технологии**: Go, Gin, GORM, PostgreSQL с расширением PostGIS
- **Структура данных**:
  - Железнодорожные станции с координатами
  - Районы, линии, области

## FileHandler
- **Описание**: Компонент для обработки CSV файлов
- **Основные функции**:
  - Парсинг CSV файлов с метеостанциями
  - Отправка данных в API метеостанций
  - Обработка ошибок и логирование
- **Формат данных**: CSV с разделителем ";"
- **Обработка**: По строкам с пропуском заголовка

## Технологии
- **Язык программирования**: Go
- **Веб-фреймворк**: Gin
- **ORM**: GORM
- **Базы данных**: PostgreSQL, TimescaleDB, PostGIS
- **Документация API**: Swagger
- **Геохеши**: Для географической привязки данных

## Взаимодействие микросервисов
1. Пользователь через веб-интерфейс выбирает железнодорожную станцию и временной интервал, отправляет запрос на /api/railway/analyze.
2. API Gateway перенаправляет запрос в Railway Service.
3. Railway Service извлекает координаты станции из своей БД.
4. Railway Service вызывает Weather Station Service (/stations/nearby) с этими координатами.
5. Weather Station Service отправляет список трёх ближайших метеостанций.
6. Railway Service вызывает Weather Data Service (/aggregate) с идентификаторами этих станций и интервалом дат.
7. Weather Data Service отправляет агрегированные данные по опасным явлениям.
8. Railway Service объединяет информацию и возвращает результат.
9. API Gateway отдаёт ответ клиенту.
10. При повторном идентичном запросе (в пределах TTL) результат извлекается из Redis, минуя обращение к нижележащим сервисам.

## Недостающие компоненты

### 1. API Gateway
- **Описание**: Точка входа для всех запросов от клиента.
- **Технологии**: nginx
- **Функции**:
  - Принятие запросов от веб-интерфейса
  - Маршрутизация запросов к соответствующим сервисам
  - Агрегирование ответов

### 2. Сервис метеоданных (Weather Data Service)
- **Описание**: Отвечает за запись и чтение первичных метеорологических наблюдений, получение агрегированных данных по опасным явлениям.
- **Технологии**: Go, Gin, GORM, TimescaleDB
- **Функции**:
  - Проверка Redis
  - Загрузка активных правил из БД
  - Выполнение запросов к таблице наблюдений для подсчёта числа случаев за указанный интервал
  - Агрегирование результатов, кэширование и возвращение

### 3. Сервис метеостанций (WeatherStationService)
- **Описание**: Хранит справочную информацию о метеостанциях.
- **Технологии**: Go, Gin, GORM, PostgreSQL с PostGIS
- **Функции**:
  - Проверка Redis
  - Выполнение пространственного запроса к PostGIS
  - Кэширование результата и возврат

### 4. Сервис железнодорожных объектов (Railway Service)
- **Описание**: Хранит данные о станциях, линиях.
- **Технологии**: Go, Gin, GORM, PostgreSQL с PostGIS
- **Функции**:
  - Проверка наличия результата в Redis
  - Извлечение координат станции из БД
  - Вызов Weather Station Service для получения трёх ближайших метеостанций
  - Вызов Weather Data Service с перечнем идентификаторов метеостанций и интервалом дат
  - Объединение ответов и сохранение в Redis
  - Отправка результата клиенту

### 5. Сервис визуализации (Visualization Service)
- **Описание**: Перенаправляет данные для фронтенда.
- **Технологии**: Go, Gin, GORM
- **Функции**:
  - Передача данных для GeoJSON карт, JSON таблиц и графиков

## Запуск сервисов
1. Запустить базу данных PostgreSQL
2. Настроить переменные окружения в .env файлах
3. Запустить каждый микросервис отдельно:
   - Meteostation: `go run Meteostation/main.go`
   - Metedata: `go run Metedata/main.go`
   - RailWayStation: `go run RailWayStation/main.go`
   - FileHandler: `go run FileHandler/main.go`

## Порты
- Meteostation: 8081
- Metedata: 8083
- RailWayStation: 8082
- FileHandler: 8080 (по умолчанию)
